## 二、知识库设计原则

### 为什么按主题分文件？

1. **Dify RAG 最佳实践**：每个文件聚焦一个主题，检索时精准匹配相关文件，避免跨主题信息干扰
2. **独立自包含**：每个文件可独立回答该主题下的问题，不需要引用其他文件
3. **更新方便**：价格变动只改一个文件，FAQ 新增只改一个文件
4. **按需加载**：不同场景（官网/小程序/公众号）可选择性地导入不同文件

### 知识颗粒度建议

- 每个文件控制在 **100-200 行**左右，适合 LLM 上下文窗口
- 标题层级清晰（H1→H2→H3），便于分段检索
- 使用表格呈现结构化数据，便于 AI 准确提取
- FAQ 文件使用 Q&A 格式，问答一一对应

## 三、导入 Dify 知识库

### 步骤

1. 登录 Dify 平台 → 知识库 → 创建知识库
2. 命名：`流云智炬业务知识库`
3. 上传方式：选择"批量导入 Markdown 文件"
4. 将 `AI知识库/` 下所有 `.md` 文件拖入上传
5. 分段设置：按 Markdown 标题自动分段（H2 为分段边界）
6. 索引方式：高质量模式（Embedding）
7. 等待向量化完成

### 分段策略建议（Dify）

| 参数 | 建议值 | 说明 |
|------|:---:|------|
| 分段最大长度 | 800 tokens | 每个分段足够回答一个问题 |
| 分段重叠 | 50 tokens | 避免边界信息丢失 |
| 分隔符 | `##`（H2标题） | 以二级标题为自然分段点 |
| 检索方式 | 混合检索（向量+关键词） | 提高匹配准确率 |

## 四、用于微信小程序 AI 客服

### 方式一：Dify API 对接

小程序前端 → 调用 Dify Chat API → Dify 自动检索知识库 → 返回 AI 答案

```
wx.request({
  url: 'https://your-dify-instance.com/v1/chat-messages',
  method: 'POST',
  data: {
    inputs: {},
    query: userMessage,
    response_mode: 'blocking',
    user: userId
  },
  headers: {
    'Authorization': 'Bearer YOUR_DIFY_API_KEY'
  }
})
```

### 方式二：CloudBase AI 对接

如果使用微信云开发，可将 Markdown 知识库上传到 CloudBase 云存储，通过云函数调用 AI 接口 + 知识检索。

## 六、竞品对比话术（AI可用）

当客户询问"你们和XX比怎么样"时，AI可参考以下核心话术：

**vs SaaS 平台（凡科/上线了）**："我们一次性付费，源码归你，搬家自由。SaaS 是年费制，3年下来比我们贵，而且源码不归你。"

**vs 淘宝模板站**："淘宝买的是租的房子，我们建的是你自己的房子。"

**vs 一线城市团队**："同品质服务，我们便宜30-50%。核心团队来自一线互联网公司，能力一样，成本结构不同。"

**一句话定位**："比一线团队便宜50%，比SaaS平台更自主——海南本地一站式数字化伙伴。"
