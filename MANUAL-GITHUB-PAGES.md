# 🔧 GitHub Pages 手动设置指南
## 📋 当你需要手动启用GitHub Pages时

### 🎯 快速设置步骤

1. **打开设置页面**
   - 访问：`https://github.com/TO3C/lyzjkj/settings/pages`
   - 确认你已登录到正确的账号

2. **进入Pages设置**
   - 在设置页面中点击"Settings"选项卡
   - 在左侧菜单中找到"Pages"选项
   - 点击进入Pages设置页面

3. **部署设置**
   - 在"Source"部分选择"Deploy from a branch"
   - 在"Branch"下拉菜单中选择"main"分支
   - 在"/"目录选择"根目录"/（推荐）
   - 点击"Save"按钮保存设置

### 🎯 检查部署状态

设置完成后，GitHub会自动：
1. **开始构建**：通常需要1-2分钟
2. **部署网站**：构建完成后自动发布
3. **提供访问链接**：在设置页面中显示网站地址

### 📊 网站地址

成功部署后，你的网站将在以下地址可访问：
- **GitHub Pages地址**：`https://TO3C/lyzjkj.github.io`
- **或自定义域名**：如果设置了的话

### 📋 监控构建

你可以在设置页面中：
- 查看构建日志
- 查看部署历史
- 查看访问统计

### 📋 常见问题及解决方案

如果遇到部署失败，可能的原因：
- **构建错误**：代码语法错误或依赖问题
- **分支问题**：选择了错误的分支
- **权限问题**：没有足够的仓库访问权限
- **设置错误**：选择了错误的目录或设置

### 🎯 成功标志

当设置页面显示以下状态时，表示设置成功：
- **✅ 绿色指示器**："Your site is published at..."
- **成功消息**："Your site was successfully published"

## 📋 设置完成后

完成设置后，请：
1. **等待1-2分钟**：让GitHub完成构建
2. **刷新网站**：在几分钟后访问你的网站
3. **测试功能**：确认所有功能正常工作
